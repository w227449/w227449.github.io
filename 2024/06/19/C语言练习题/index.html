B) !(c<=48||c>=57&&c<=65||c>=90&&c<=97||c>=122)
C) c>=48&&c<=57||c>=65&&c<=90||c>=97&&c<=122
D) !(c>=48&&c<=57||c>=65&&c<=90||c>=97&&c<=122)

答案：D

2、有以下程序

#include <stdio.h>
main()
{ int y=10;
while(y--); printf("y=%d\n", y);
}
程序执行后的输出结果是（ ）。
A) y=0 B) y=-1 
 C) y=1 D) while构成无限循环

答案：B

    解析：这段程序中，while(y--); 构成了一个循环。
    这个循环的条件是 y--，
    这是一个后置递减表达式，
    意味着每次循环迭代时 y 的值会先被使用，
    然后 y 的值会减少1。
    初始时，y 的值为10。
    在循环中，y 会递减直到变为0。
    由于 while 循环的条件是 y--，
    当 y 变为0时，
    循环条件不再满足，循环结束。
    但是，由于 y-- 是后置递减，
    所以即使 y 变为0，
    循环条件 y-- 会先使用 y 的当前值（0），
    然后 y 会减1，变为-1。
    循环结束后，y 的值变为-1。然后执行 printf 语句，输出 y 的值。
    所以程序执行后的输出结果是 y=-1，正确答案是 B y=-1

3、设有定义：int x＝2；，以下表达式中，值不为6的是( )。

A) 2*x,x+=2 B) x＋＋，2*x
C) x*＝(1＋x) D) x*＝x＋1

答案：A

解析：
A) 2*x, x += 2 首先计算2*x，此时x的值为2，所以结果为4。然后执行x += 2，将x的值增加2，变为4。
B) x++, 2*x 首先执行x++，x的原始值为2，使用这个值后x增加1变为3。然后计算2*x，此时x的值为3，所以结果为6。
C) x *= (1 + x) 首先计算1 + x，此时x的值为2，所以结果为3。然后执行x *= 3，将x的值乘以3，变为6。
D) x *= x + 1 首先计算x + 1，此时x的值为2，所以结果为3。然后执行x *= 3，将x的值乘以3，变为6

4、以下定义数组的语句中错误的是( )。

A) int num[][3]＝{ {1,2},3,4,5,6 }；
B) int num[2][4]＝{ {1,2}，{3,4}，{5,6} }；
C) int num[]＝{ 1,2,3,4,5,6 }；
D) int num[][4]＝{1,2,3,4,5,6}；

答案：B

A) int num[][3] = { {1, 2}, 3, 4, 5, 6 };
这个语句试图定义一个二维数组，其中第二维的大小为3。数组初始化列表中包含了6个元素，可以被分为两个长度为3的子数组，这与定义的数组维度相匹配，因此这个语句是正确的。

B) int num[2][4] = { {1, 2}, {3, 4}, {5, 6} };
这个语句试图定义一个2行4列的二维数组。但是初始化列表中只包含了3个元素，每个元素都是一个包含两个元素的数组。这与定义的数组维度不匹配，因为2行4列的数组需要8个元素。因此，这个语句是错误的。

C) int num[] = { 1, 2, 3, 4, 5, 6 };
这个语句定义了一个一维数组，其中包含了6个元素。由于数组的大小没有在初始化时指定，编译器会根据初始化列表中的元素数量来确定数组的大小。这个语句是正确的。

D) int num[][4] = {1, 2, 3, 4, 5, 6};
这个语句定义了一个二维数组，其中第二维的大小为4。初始化列表中包含了6个元素，可以被分为两个长度为4的子数组，这与定义的数组维度相匹配，因此这个语句是正确的。

5、若有说明：int st[5]； 则对st 数组元素的正确引用是（ ）。

    A) st[5 - 1 ]  B) st[3.0] 
    C) st[2 + 3 ]  D) st( 1 )

答案：A

解析：
A) st[5 - 1]
这个表达式是正确的，因为 5 - 1 的结果是4，这是数组 st 的有效索引。
B) st[3.0]
这个表达式是错误的，因为数组索引必须是整数值，而3.0是一个浮点数。
C) st[2 + 3]
这个表达式是正确的，因为 2 + 3 的结果是5，但是索引5超出了数组 st 的有效范围（0到4），所以这个引用在运行时会导致数组越界。
D) st(1)
这个表达式是错误的，因为数组的引用应该使用方括号 [] 而不是圆括号 ()

    { int a＝0，b＝0，c＝0，d＝0；
     if(a＝1) b＝1；c＝2；

所以，正确答案是 D 编译有错。

7、若有语句：int a=4, *p=&a；下面均代表地址的一组选项是（ ）。

    A) a,p,&*a B) &a,p,&*p 
    C) *&a,&a,*p D) *&p,*p,&a

答案：B

解析：
a 是变量的值，不是地址。
p 是指针变量，它存储的是变量 a 的地址。
&*a 是先取 a 的地址，然后解引用该地址，得到 a 的值，不是地址。
*&a 是先取 a 的地址，然后解引用该地址，得到 a 的值，不是地址。
&a 是变量 a 的地址。
*p 是解引用指针 p，得到 a 的值，不是地址
*&p 是先取指针 p 的地址，然后解引用这个地址，这将得到 p 指针本身的值，即变量 a 的地址，这是一个地址。
*p 是解引用指针 p，得到 a 的值，不是地址。
&a 是变量 a 的地址。

8、设变量a是int型，f是float型，i是double型，则表达式10+’a’+i*f值的数据类型为（ ）。

A) char         B) int 
C) float        D) double

答案：D

解析：
char 和 short int 类型的值在参与运算前会被提升到 int 类型。
如果运算中存在 float 类型的值，int 类型的值会被提升到 float 类型。
如果运算中存在 double 类型的值，其他数值类型的值都会被提升到 double 类型。
如果有更复杂的类型混合，比如 long int、long float 等，会根据它们在类型提升层级中的位置来确定最终的运算结果类型。

9、下面程序的运行结果是（ ）。

printf ("%d , %d, %d", a, b, c)；

A) 2, 1, 0 B) 2, 1, 1
C) 1, 2, 0 D) 1, 2, 1
答案：C
解析：一开始满足循环条件，因此进入循环，a，b调换值，执行循环过程中c值不变，此时c值为2，因此再次进入循环，循环进行中c为1，不符合再次进入循环条件，跳出循环，循环结束后c为0，因此a，b值再次调换，即为原值。

15、有以下程序：

#include <stdio.h>
main()
{ char a，b，c，d；
scanf("%c%c"，&a， &b)；
c＝getchar()； d＝getchar()；
printf("%c%c%c%c\n"，a，b，c，d)；
}
当执行程序时，按下列方式输入数据(从第1列开始，<CR>代表回车，注意：回车也是一个字符)
12<CR>
34<CR>
则输出结果是( )。

A) 12 B) 1234
C) 1234 D) 123
答案：D
这段程序的目的是读取两个字符并存储在变量 a 和 b 中，然后使用 getchar() 函数读取两个字符并存储在变量 c 和 d 中。最后，程序打印出这四个字符。
在 scanf 函数中，%c 格式说明符用于读取字符，包括空格和制表符。然而，scanf 在读取完 a 和 b 后，会留下一个换行符 （回车符）在输入缓冲区中，因为用户在输入 “12” 后按了回车键。
接下来，getchar() 被调用两次。第一次调用 getchar() 会读取并丢弃掉输入缓冲区中的换行符 ，然后第二次调用 getchar() 会读取下一个字符 ‘3’ 并将其存储在变量 c 中。第三次调用 getchar() 读取下一个字符 ‘4’ 并将其存储在变量 d 中。
因此，当输入 “12” 后按回车，程序实际上读取了 ‘1’，’2’，然后是换行符 （被 getchar() 读取并丢弃），接着是 ‘3’ 和 ‘4’。输出将会是 ‘1’，换行，然后是 ‘2’，换行，接着是 ‘3’。

20、以下程序段运行的结果是（ ）。

char str[ ] ={ 'H', 'a', '\0', 'p', '\0', 'p','y' }；
printf("%s", str )；

A) 'H', 'a'     B) Ha 
C) Happy        D) Ha p py

答案：B
解析：
在 C 语言中，printf 函数使用 %s 格式说明符打印字符串时，会从指定的字符数组开始打印，直到遇到空字符 ‘\0’（字符串的结束标记）。在提供的程序段中，字符数组 str 包含了 ‘H’, ‘a’, ‘\0’, ‘p’, ‘\0’, ‘p’, ‘y’。这里的 ‘\0’ 表示字符串的结束。
由于 ‘\0’ 出现在 ‘a’ 之后，所以 printf 函数在打印 ‘a’ 后遇到 ‘\0’ 就会停止打印，因此输出的字符串是 “Ha”。

22、设有以下定义：

int a=0； 
double b=1.25；
char c='A'；
#define d 2

则下面语句中错误的是（ ）。

A) a++； B) b++； 
C) c++； D) d++；

答案：D

解析：
在 C 语言中，++ 运算符可以用于整型和浮点型变量，以进行递增操作。对于选项 A) a++ 和 B) b++，由于 a 是 int 类型，b 是 double 类型，它们都可以使用 ++ 运算符。
对于 C) c++，虽然 c 是 char 类型，但 char 类型在 C 语言中可以隐式转换为 int 类型，因此 c++ 也是合法的，并且会将字符 c 的 ASCII 值加 1。
然而，对于 D) d++，d 是一个宏定义，它不是一个变量，因此不能对其进行递增操作。宏定义只是预处理指令，用于在编译前替换文本，它们没有类型，也没有存储位置，所以不能对它们使用 ++ 运算符。  
因此，错误的语句是 D) d++。

24、以下对结构体变量stu中成员的正确引用是( ) 。

struct student
{ int num；
 struct date 
 { int month；
int day；
int year；
} birthday；
}stu；

A) month B) stu.day
C) stu.birthday D) stu.birthday.year
答案：D

在 C 语言中，结构体变量的成员可以通过点号（.）来引用。对于嵌套结构体，需要连续使用点号来访问最内层的成员。
这里，stu 是 student 类型的结构体变量，它包含一个名为 birthday 的 date 类型成员。birthday 又包含 month、day 和 year 三个成员。
选项 A) month 是 date 结构体的成员，但没有指定它属于哪个 date 实例，因此不正确。
选项 B) stu.day 试图访问 day 成员，但 day 是 date 结构体的成员，所以应该通过 stu.birthday.day 来访问。
选项 C) stu.birthday 正确地引用了 birthday 成员，但它本身是一个 date 类型的结构体，而不是一个单独的成员。
选项 D) stu.birthday.year 正确地引用了 birthday 成员中的 year 成员。
因此，正确答案是 D) stu.birthday.year。
总而言之类似与电脑文件夹，要一层一层引用

25、有定义如下：

struct sk
{ int a；
float b；
}data ,*p；
如果 p=&data； 则对于结构体变量data的成员a的正确引用是（ ）。
    A) (*p).a        B)   p->data.a 
    C) p.data.a      D) (*).data.a

答案：A

在 C 语言中，结构体变量的成员可以通过两种方式引用：
1.使用点号（.）：这种方式适用于直接访问结构体变量的成员。
2.使用箭头（->）：这种方式适用于通过结构体指针访问结构体的成员。
给定的结构体定义和指针初始化如下：
struct sk
{ int a;
  float b;
} data ,*p;
p = &data;
这里，data 是 sk 类型的结构体变量，p 是指向 sk 类型结构体的指针，并且已经被初始化为指向 data。
选项 A) (*p).a 使用了去引用操作 (*p) 来访问指针 p 指向的结构体的成员 a。这是正确的引用方式。
选项 B) p->data.a 试图使用箭头操作符 -> 来访问成员，但是 data 不是指针 p 指向的结构体的成员，所以这是错误的。
选项 C) p.data.a 试图直接通过指针 p 访问成员 data.a，但是 data 不是指针 p 的成员，所以这是错误的。
选项 D) (*).data.a 这种写法没有意义，因为它试图对一个不明确的表达式去引用，并且 data.a 也不是一个有效的成员引用。

26、以下合法的字符串常量的是（ ）。

    A) ‘F’      B) "" 
    C)‘ABCD’    D) abc

答案：B

     在 C 语言中，字符串常量是由双引号 " " 包围的字符序列，而字符常量是由单引号 ' ' 包围的单个字符。

    选项 B) "" 是一个空的字符串常量，它是合法的字符串常量。
    选项 C) 'ABCD' 实际上是四个连续的字符常量，而不是一个字符串常量。正确的字符串常量应该是 "ABCD"。
    选项 D) abc 没有使用任何引号，因此它不是字符串常量也不是字符常量。如果它被双引号包围，如 "abc"，它将是一个字符串常量。

28、一个C程序的执行是从（ ）。

32、以下正确的函数形式是（ ）。

A) double fun1(int x,int y) 
{ z=x+y；
return z； 
}
B) fun1(int x, y) 
{ int z；
return z；
}
C) fun1(x,y) 
{ int x,y；
double z；
z=x+y；
return z；
}
D) double fun1(int x, int y) 
{ double z；
z=x+y；
return z；
}

答案：D

A) double fun1(int x,int y) { z=x+y; return z; }
这个函数定义缺少了变量 z 的声明。在函数体中使用未声明的变量会导致编译错误。
B) fun1(int x, y) { int z; return z; }
这个函数定义缺少了第二个参数 y 的类型。在 C 语言中，除非函数之前已经声明过，否则每个参数都需要指定类型。
C) fun1(x,y) { int x,y; double z; z=x+y; return z; }
这个函数定义中，参数列表中的 x 和 y 被重新声明为局部变量，这会覆盖原来的参数。此外，函数的返回类型未指定，这会导致编译错误。
D) double fun1(int x, int y) { double z; z=x+y; return z; }
这个函数定义是正确的。它具有正确的返回类型 double，参数列表正确地声明了两个 int 类型的参数 x 和 y，函数体中声明了 double 类型的局部变量 z 并返回了它的值。
因此，正确答案是 D) double fun1(int x, int y) { double z; z=x+y; return z; }。

B) fopen(“test.txt”, “r+”);
这个调用以读写模式打开文件，如果文件存在，可以读取和修改它；如果文件不存在，将导致错误
C) fopen(“test.txt”,”rb”);
这个调用以二进制模式打开文件进行只读操作，通常用于二进制文件，而不是文本文件。
D) fopen(“test.txt”, “w”);
这个调用以写模式打开文件，如果文件存在，它会被截断为零长度，如果文件不存在，将会创建新文件。但是，这个模式不允许读取文件。

43、若x是int型变量，且有下面的程序片段：

    for(x＝3；x<6；x＋＋) printf((x%2)？("**%d")：("##%d\n")，x)；

上面程序片段的输出结果是( )。

    A)  **3
        ##4
        **5
    B)  ##3
        **4
        ##5
    C)  ##3
        **4
        ##5
    D)  **3
        ##4
        **5

答案：B
在这个循环中，x 的初始值是 3。循环条件 x < 6 确保了循环会执行三次，直到 x 等于 5。x++ 是循环的增量表达式，这意味着每次循环迭代结束时 x 的值会增加 1。
printf 函数中的三元运算符 (? :) 根据 x % 2 的结果来决定使用哪个格式字符串：
如果 x % 2 结果为非零（即 x 是奇数），则使用 “**%d” 作为格式字符串。
如果 x % 2 结果为零（即 x 是偶数），则使用 “##%d\n” 作为格式字符串。
当 x 是 3（奇数），printf 将打印 “**3”。
当 x 是 4（偶数），printf 将打印 “##4\n”，并且由于格式字符串中的 \n，输出将在新的一行开始。
当 x 是 5（奇数），printf 将打印 “**5”。
因此，修正后的程序片段的输出结果是：
**3
##4
**5

45、有以下程序：

    #include <stdio.h>
    main( )
    { char c1，c2，c3，c4，c5，c6；
    scanf("%c%c%c%c"，&c1，&c2，&c3，&c4)；
    c5＝getchar()； c6＝getchar()；
    putchar(c1)；putchar(c2)；
    printf("%c%c\n"，c5，c6)；
    }
    程序运行后，若从键盘输入(从第1列开始) 
    123<回车>
    45678<回车>

则输出结果是( )。
A) 1267
B) 1256
C) 1278
D) 1245
答案：D
解析：
让我们分析程序的行为：
程序定义了六个 char 类型的变量：c1 到 c6。
使用 scanf 函数读取四个字符并存储在 c1 到 c4 中。scanf 的格式字符串 “%c%c%c%c” 指定了四个 %c，每个 %c 将匹配一个字符。
scanf 会读取输入直到遇到空白字符（空格、制表符或换行符）。因此，输入的 “123” 中的字符 ‘1’、’2’、’3’ 将分别赋给 c1、c2 和 c3。由于 scanf 会停止在空白字符前，所以第四个 %c 不会读取字符，c4 将不会被赋值。
getchar() 读取一个字符，丢弃掉 scanf 留下的换行符（<回车>），然后存储读取的下一个字符到 c5。由于 scanf 停止在 ‘3’ 后，getchar() 将读取 ‘4’。
再次调用 getchar() 读取并存储下一个字符到 c6，即 ‘5’。
putchar(c1); putchar(c2); 将打印 c1 和 c2 的值，即 ‘1’ 和 ‘2’。
printf(“%c%c\n”, c5, c6); 将打印 c5 和 c6 的值，即 ‘4’ 和 ‘5’，后跟一个换行符。
因此，程序的输出将是：
12
45

46、下列语句组中，正确的是( )。

    A) char *s；s＝"Olympic"；      B) char s[7]；s＝"Olympic"；
    C) char *s；s＝{"Olympic"}；     D) char s[7]；s＝{"Olympic"}；

答案：A
解析：
A) char *s; s = “Olympic”;
这个语句是正确的。这里 s 是一个指向 char 的指针，它被赋值为字符串字面量 “Olympic” 的地址。这个字符串字面量在内存中的位置是常量，所以这个赋值操作是合法的。
B) char s[7]; s = “Olympic”;
这个语句是错误的。尽管数组 s 的大小足以存储 “Olympic”（包括结尾的空字符 ‘\0’），但是不能直接用字符串字面量来赋值给字符数组。正确的方式是使用字符串复制函数 strcpy(s, “Olympic”);。
C) char *s; s = {“Olympic”};
这个语句是错误的。在 C 语言中，没有使用 {} 来对字符串字面量进行初始化或赋值的语法。
D) char s[7]; s = {“Olympic”};
这个语句也是错误的。同样，{} 不能用于字符串字面量的赋值，而且即使去掉 {}，也不能直接将字符串字面量赋值给字符数组，如选项 B 所述。

52、以下选项中，合法的一组C语言数值常量是( )。

    A) 12. 0Xa23 4.5e0
    B) 028 .5e－3 －0xf
    C) .177 4e1.5 0abc
    D) 0x8A 10,000 3.e5

答案：A

    解析：
    A) 12. 0Xa23 4.5e0
    12. 是一个合法的浮点数。
    0Xa23 是一个合法的十六进制整数（注意：通常十六进制数以 0x 或 0X 开头）。
    4.5e0 是一个合法的科学计数法表示的浮点数。
    B) 028 .5e－3 －0xf
    028 是一个合法的八进制数。
    .5e－3 缺少数字前的零，应该是 0.5e-3。
    －0xf 缺少负号前的数字，应该是 -0xf。
    C) .177 4e1.5 0abc
    .177 缺少数字前的零，应该是 0.177。
    4e1.5 科学计数法中不能有小数点，应该是 4e1 或 4.5e1。
    0abc 不合法，因为 C 语言中没有以 0x 或 0X 开头的十进制数。
    D) 0x8A 10,000 3.e5
    0x8A 是一个合法的十六进制整数。
    10,000 在 C 语言中，数值常量中不能包含逗号。
    3.e5 是一个合法的科学计数法表示的浮点数。
